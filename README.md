# Simulação Educacional de Phishing em Ambiente Virtualizado

## Sobre o Projeto

Este projeto foi desenvolvido como atividade prática da disciplina de **Administração de Servidores**, com foco no estudo técnico e crítico de mecanismos utilizados em ataques de **phishing**.

A proposta consistiu na criação de uma **página web que simula o clone de uma aplicação popular/profissional da internet**, com o objetivo de compreender, em ambiente controlado, como esse tipo de golpe pode ser estruturado e quais fatores tornam os usuários vulneráveis.

A aplicação foi desenvolvida para fins exclusivamente **acadêmicos**, permitindo a análise prática de:

- clonagem visual de interfaces web;
- captura simulada de credenciais;
- armazenamento de dados em banco de dados;
- hospedagem em servidor web Apache;
- transferência de arquivos via FTP;
- configuração de DNS local para simulação de domínio funcional.

> **Aviso importante:** este projeto foi desenvolvido exclusivamente para fins educacionais, em **máquinas virtuais** e **ambiente isolado**, sem qualquer finalidade maliciosa.

---

## Objetivo

O principal objetivo deste projeto foi proporcionar uma experiência prática sobre:

- **Segurança da Informação**
- **Administração de Servidores**
- **Hospedagem de aplicações web**
- **Transferência de arquivos em rede**
- **Configuração de DNS local**
- **Conscientização sobre engenharia social e phishing**

## Ambiente de Execução

O projeto foi implementado em um ambiente virtualizado, utilizando:

- **Máquina local** para desenvolvimento da aplicação
- **Servidor Ubuntu Server** para hospedagem
- **Servidor web Apache**
- **Banco de dados** para armazenamento das credenciais simuladas
- **Configuração de DNS local**, simulando um domínio real

Exemplo de domínio utilizado em laboratório:
- `linkedin.local`

---

## Funcionalidades Implementadas

- Interface web inspirada em uma plataforma profissional popular
- Formulário para inserção de credenciais
- Captura de dados enviados pelo formulário
- Armazenamento dos dados em banco de dados
- Hospedagem da aplicação em servidor Apache
- Transferência dos arquivos via FTP
- Acesso via DNS local configurado no ambiente virtualizado
